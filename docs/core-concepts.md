# Core Concepts

## Input

Any work entering the framework.

Examples:

- Feature request
- Production bug
- Security issue
- Technical debt
- Documentation update

Input does not define priority.

Input simply identifies work.

---

## Development

The activity of producing or modifying software.

Development may be performed by:

- A developer
- Multiple developers
- Pair programming
- AI
- Any combination

---

## Validation

Independent verification that the work is ready to ship.

Validation may be:

- Manual
- Automated
- Code review
- Testing
- Static analysis

---

## Ship

Making validated work available to its intended users.

How software is shipped is implementation detail.
