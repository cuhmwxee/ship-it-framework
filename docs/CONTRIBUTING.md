# Contributing

Thanks for your interest in the Ship It! Framework.

## Philosophy

Keep things simple.

Every contribution should improve clarity, not complexity.

## Workflow

Input

↓

Development

↓

Validation

↓

Ship

## Guidelines

* Keep discussions respectful.
* Prefer small pull requests.
* One feature per pull request.
* Keep the framework minimal.
* Avoid unnecessary complexity.

If you're unsure, open an issue before implementing a larger change.
